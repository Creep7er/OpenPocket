# PopugVPocket 0.5.1

PopugVPocket 0.5.1 is a stabilization and distribution-infrastructure release following Reborn.

## Highlights

- VGirl now uses independent left direction, center display, and right action zones across common landscape sizes.
- XYAB touch rectangles no longer overlap and switch between compact and diamond layouts responsively.
- Store downloads expose progress, cancellation, bounded redirects and size, `.part` staging, SHA-256 verification, and atomic completion.
- Store UI supports update, reinstall, repair, verify, and uninstall while preserving achievements and permanent rewards.
- Android workflows can build compact APK, AAB, checksums, and either debug-prerelease or production-signed artifacts.
- Cartridge templates generate release assets and a moderated catalog entry draft.

## Android

- Package: `org.popugonet.popugvpocket`
- versionName: `0.5.1`
- versionCode: `7`
- Minimum SDK: 29
- VBoy portrait and VGirl landscape profiles

The public 0.5.1 release is a debug prerelease intended for testing. Tagged releases currently publish a debug-signed APK and checksum without an AAB. Production mode remains available manually, requires repository signing secrets, and includes no production keystore in the repository.

## Security

External cartridges remain unsandboxed Godot code. HTTPS and SHA-256 validate expected bytes but not publisher identity. Install only cartridges whose source and publisher you trust.

## Known limitations

- The public catalog has no approved external GitHub Release assets yet; production remote installation is not claimed as verified.
- No Android device was available during the initial 0.5.1 preparation pass.
- Updating a mounted PCK may require restarting PopugVPocket.

Checksums are generated as `SHA256SUMS.txt` beside release artifacts.
